Ejercicio 1 · POO básica y estructuras de control

Clase, atributos privados, constructor, getters/setters, if-else y bucle for
2,5 puntos

Crea la clase Pelicula con los atributos privados titulo (String), director (String), anyoEstreno (int) y duracionMinutos (int).

  1. a) Constructor con los cuatro parámetros. Getters y setters para cada atributo. [1,0 pt]
  2. b) Método mostrarFicha() que imprima la información con el formato:
    [año] Título — Director (XXX min)
    [0,5 pts]
  3. c) En el main: crea dos objetos Pelicula y usa if-else para mostrar cuál tiene mayor duración. Si duran lo mismo, imprime "Misma duración". [0,5 pts]
  4. d) Con un bucle for, clasifica la película según su duración mostrando una etiqueta diferente para cada rango:
    • Menos de 60 minutos → "Cortometraje"
    • Entre 60 y 90 minutos → "Película corta"
    • Entre 91 y 120 minutos → "Duración estándar"
    • Más de 120 minutos → "Película larga"
    Aplica la clasificación a cada película del array creado en el apartado c) e imprime la etiqueta junto a su título. [0,5 pts]

Ejercicio 2 · Interfaz Catalogable y herencia

Interfaz, implementación, herencia, sobrescritura y polimorfismo
2,5 puntos

El sistema de la videoteca necesita estandarizar cómo se muestran los objetos en el catálogo.

  1. a) Define la interfaz Catalogable con dos métodos:
    • String getCodigo()
    • void mostrarEnCatalogo()
    (Nota: mostrarEnCatalogo() imprime por consola, no devuelve String) [0,5 pts]
  2. b) Haz que Pelicula implemente Catalogable.
    • El código se forma concatenando "PEL-", el año de estreno y las tres primeras letras del título en mayúsculas.
    • El método mostrarEnCatalogo() llamará a mostrarFicha() y añadirá el código entre corchetes al final.
    [2019] El Irlandés — Scorsese (209 min) [PEL-2019-ELI]
    [0,75 pts]
  3. c) Crea PeliculaAnimacion que herede de Pelicula y añada el atributo privado estudio (String, p. ej. "Pixar"). Añade constructor con super(), getter/setter y sobreescribe mostrarEnCatalogo() para mostrar también el estudio de animación. [0,75 pts]
  4. d) En el main: crea una Pelicula y una PeliculaAnimacion, guárdalas en un array Catalogable[] y recórrelo llamando a mostrarEnCatalogo(). Añade un comentario explicando qué concepto OOP demuestra. [0,5 pts]

Ejercicio 3 · Gestión con ArrayList e Interfaces

ArrayList, interfaz funcional, búsqueda, filtrado y ordenación
2,5 puntos

El sistema necesita gestionar colecciones de películas de forma eficiente usando listas dinámicas.

  1. a) Define la interfaz Gestionable<T> con los siguientes métodos:
    • void agregar(T elemento)
    • boolean eliminar(T elemento)
    • T buscarPorTitulo(String titulo)
    • void mostrarTodos()
    [0,5 pts]
  2. b) Crea la clase Videoteca que implemente Gestionable<Pelicula> usando internamente un ArrayList<Pelicula>.
    • agregar(Pelicula p) añade la película si no existe ya una con el mismo título (ignora mayúsculas). Si ya existe, lanza IllegalArgumentException con un mensaje descriptivo.
    • eliminar(Pelicula p) elimina la película por referencia y devuelve true si se eliminó, false en caso contrario.
    • buscarPorTitulo(String titulo) devuelve la primera película cuyo título contenga la cadena buscada (sin distinguir mayúsculas) o null si no se encuentra.
    • mostrarTodos() llama a mostrarFicha() de cada película.
    [1,0 pt]
  3. c) Añade a Videoteca el método List<Pelicula> filtrarPorDirector(String director) que devuelva un nuevo ArrayList solo con las películas de ese director (sin distinguir mayúsculas). [0,5 pts]
  4. d) En el main:
    • Crea una Videoteca y añade al menos 5 películas.
    • Intenta añadir una película duplicada y captura la excepción mostrando el mensaje.
    • Busca una película por título parcial e imprime su ficha si se encuentra.
    • Filtra por director e imprime las películas resultantes.
    [0,5 pts]

Ejercicio 4 · Streams, Listas e Iterator

Stream API, List, Iterator y recorrido de colecciones
2,5 puntos

El sistema necesita analizar, recorrer y transformar la colección de películas de la Videoteca del ejercicio anterior usando distintos mecanismos de Java.

  1. a) Añade a Videoteca el método List<Pelicula> getPeliculas() que devuelva la lista interna de películas. A continuación, en el main, recorre esa lista de tres formas distintas imprimiendo el título de cada película en cada caso:
    • Con un bucle for clásico usando índice.
    • Con un bucle for-each.
    • Con un Iterator<Pelicula> obtenido mediante iterator().
    Añade un comentario explicando en qué situación es preferible usar Iterator frente a un for-each. [0,75 pts]
  2. b) Usa un Iterator para eliminar de la lista todas las películas con una duración inferior a 90 minutos, sin lanzar ConcurrentModificationException. Imprime la lista resultante usando mostrarFicha(). [0,5 pts]
  3. c) A partir de la lista de películas de la Videoteca, usa la Stream API para realizar cada una de las siguientes operaciones de forma independiente (abre un nuevo stream en cada caso):
    • Filtrar las películas estrenadas a partir del año 2000 e imprimir sus títulos (filter + forEach).
    • Obtener un List<String> con los títulos de todas las películas en mayúsculas (map + collect) e imprimirlo.
    • Calcular la duración media de todas las películas e imprimirla (mapToInt + average).
    • Ordenar las películas por año de estreno de mayor a menor con sorted, recogerlas en una nueva lista e imprimir la ficha de cada una.
    Precede cada bloque con un mensaje que identifique la operación que se está mostrando. [1,0 pt]
  4. d) Usando un Stream sobre la lista de películas, comprueba con anyMatch si existe alguna película con una duración superior a 120 minutos. Imprime "Hay películas largas" o "No hay películas largas" según el resultado.
    ¿Qué es anyMatch? Es una operación terminal de Stream que recibe una condición y devuelve true en cuanto encuentra al menos un elemento que la cumpla, o false si ninguno la cumple. Equivale a preguntar: "¿hay alguno que…?"
    El tipo de retorno de stream().anyMatch(...) es boolean, por lo que puede usarse directamente en un if.
    // Ejemplo de salida esperada Hay películas largas
    [0,25 pts]
